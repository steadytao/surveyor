Surveyor v0.8.0 adds the first workflow-oriented layer on top of the existing baseline, diffing, prioritisation and structured inventory surface. Inventory-backed audit views can now use organisational context more deliberately without turning Surveyor into a policy engine, dashboard or storage-backed governance system.

## Highlights

- Added restrained workflow controls to `surveyor diff` and `surveyor prioritize` for inventory-backed audit views, including grouping by owner, environment or source plus include filters for owner, environment and tag.
- Added metadata-aware prioritisation for inventory-backed audit input, including stronger ranking reasons and workflow findings for weak imported metadata such as missing owner, missing environment, weak provenance and inventory-port overrides.
- Added grouped workflow summaries and checked-in workflow examples so the shipped Markdown and JSON output surface now demonstrates the `v0.8.0` organisational workflow layer directly.

## Security

No known security fixes in this release.

## In this release

- This release keeps the workflow layer narrow: workflow grouping and filtering currently apply only to inventory-backed audit input or audit comparisons, and TLS input rejects those controls clearly.
- Grouped summaries now sit above the existing technical diff and prioritization models rather than replacing them, and diff-side workflow findings remain deferred even though the canonical diff report shape now includes the optional field.
- Surveyor still emits one canonical JSON output plus derived Markdown, and still does not add policy-as-code, dashboards, databases or platform-specific import adapters in this milestone.
