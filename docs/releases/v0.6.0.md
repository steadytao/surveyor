Surveyor v0.6.0 adds the first analysis layer on top of the existing inventory and audit flows. Saved reports can now be compared directly, and current reports can now be ranked for human attention without introducing a database or policy engine.

## Highlights

- Added `surveyor diff` for compatible saved `tls_scan` and `audit` reports, with canonical JSON output and derived Markdown.
- Added `surveyor prioritize` for current `tls_scan` and `audit` reports, with `surveyor prioritise` supported as a CLI alias and `migration-readiness` plus `change-risk` profiles.
- Tightened the repository docs and schema notes so the shipped baseline, diff and prioritization surface is documented honestly, including the current limits such as no `diff --profile` and no diff-input prioritisation.

## Security

No known security fixes in this release.

## In This Release

- This release keeps the analysis layer narrow: diffing currently supports compatible `tls_scan` and `audit` reports only, and prioritization currently supports current `tls_scan` and `audit` reports only.
- Current reports now carry the baseline-compatible metadata needed for saved-report comparison, and the repository includes checked-in diff and prioritization examples plus golden outputs.
