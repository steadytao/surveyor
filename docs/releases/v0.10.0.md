Surveyor v0.10.0 is the final hardening pass before v1.0.0. It does not widen the product. It makes the current TLS-first inventory, audit, diffing, prioritisation and adapter surface more stable, more reviewable and easier to trust from a clean checkout.

## Highlights

- Hardened real behaviour rather than adding breadth, including clearer diff visibility across metadata transitions, real remote attempt accounting and bounded TLS scan concurrency in the main execution paths.
- Tightened the release contract around examples, goldens and CI, with deterministic parity checks, canonical example validation and a stronger clean-checkout verification gate.
- Cleaned up the public surface so Surveyor reads more deliberately: the top-level README is now the real manual, the docs index is thinner, the main docs surface carries a consistent banner and GitHub Actions now use one bash-based shell policy.

## Security

No known security fixes in this release.

## In This Release

- `discover subnet` and `audit subnet` remain compatibility aliases, but unsupported inventory-only adapter flags are now rejected cleanly and the remote command help and default wording better match the implemented contract.
- The repository now checks more of the shipped surface directly: checked-in examples are validated against goldens and canonical headers, and the CI and release workflows exercise deterministic built-binary smoke checks instead of relying only on unit and package tests.
- Surveyor still does not add live connectors, a generic Kubernetes parser, new report kinds or broader protocol scope in this milestone. This release is about hardening and coherence, not product expansion.
