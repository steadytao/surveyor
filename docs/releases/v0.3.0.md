Surveyor v0.3.0 adds the first complete local audit workflow. The repository no longer stops at separate discovery and explicit-target TLS paths; it can now discover local endpoints, select supported TLS-like candidates conservatively and run one combined local audit with JSON-first and Markdown-derived reporting.

## Highlights

- Added `surveyor audit local`, which chains local discovery into the existing TLS scanner and emits one combined audit report.
- Added audit-side reporting that keeps discovered facts, protocol hints, scanner selection decisions, verified TLS results and explicit skip reasons visibly separate.
- Brought the README, audit docs, architecture docs, safety model, release checklist and checked-in examples into line with the implemented local audit workflow.

## Security

No known security fixes are included in this release.

## In this release

- `audit local` is local only and conservative. It only hands supported TLS-like endpoints into the existing TLS scanner.
- Discovery remains observational only. It still does not perform active probing or verified protocol identification during discovery itself.
- The explicit-target TLS inventory path remains available under `surveyor scan tls`, and its collection and classification behaviour is unchanged by this release.
