Surveyor v1.0.0 is the first supported stable release. It keeps the product narrow but it materially raises the standard of repository governance, release integrity, documentation coherence and verification around the TLS-first inventory and migration-readiness surface that Surveyor already ships.

## Highlights

- The repository surface is now substantially harder and clearer, with governance, maintainer, support, DCO, code-of-conduct and agent-instruction documents aligned at the repo root instead of scattered or implicit.
- The documentation tree is reorganised into clear command, contract, adapter, architecture, reference and release sections, with ADRs recording the durable scope and boundary decisions the project now depends on.
- The release path now publishes stronger supply-chain integrity material, including checksums, Sigstore bundles, per-archive SPDX SBOMs and provenance attestations.
- The repository now also runs OpenSSF Scorecard analysis as a dedicated supply-chain CI workflow, separate from the main build and test orchestration.

## Security

No known security fixes in this release.

## In This Release

- Surveyor still does not widen into a generic vulnerability scanner, connector platform, dashboard or storage system. This release is about hardening, coherence and trustworthy release mechanics, not product sprawl.
- The release workflow is now split deliberately: a manual preparation workflow verifies the target commit and creates the annotated tag, while the tag-triggered publishing workflow builds, signs and attests the final release assets from the immutable release ref.
- Supply-chain analysis is now broader than CodeQL and dependency automation alone. Surveyor also publishes OpenSSF Scorecard results through GitHub's security surface and the Scorecard API.
- `SECURITY.md` and `CONTRIBUTING.md` now live at the repository root, which matches GitHub's normal surface and keeps contributor-facing policy easier to discover.
