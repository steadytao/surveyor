Surveyor v0.4.1 is a narrow maintenance release focused on report-data stability. It does not widen the remote inventory surface, but it fixes an internal correctness issue so canonical reports do not retain aliases back into mutable caller-owned data.

## Highlights

- Hardened report assembly and audit-selection cloning so nested report data such as discovery hint evidence, certificate metadata and finding evidence is deep-copied before report output is built.
- Added regression coverage to keep the JSON-first and Markdown-derived reporting paths stable when upstream inputs are later mutated.
- Preserved the current `v0.4.0` command surface and release boundary; this patch is correctness hardening only.

## Security

No known security fixes in this release.

## In this release

- This release does not change `surveyor discover subnet`, `surveyor audit subnet`, `surveyor discover local`, `surveyor audit local` or `surveyor scan tls` semantics.
- The main value is internal correctness: reports now behave more like true immutable output snapshots and less like shallow wrappers around caller-owned slice data.
