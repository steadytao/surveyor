Surveyor v0.7.0 adds structured inventory inputs to the existing remote discovery and audit surface. The current remote model can now consume approved inventory manifests directly, preserve imported context and provenance through discovery and audit, and render that context in canonical JSON plus derived Markdown.

## Highlights

- Added `--inventory-file` to the canonical `surveyor discover remote` and `surveyor audit remote` commands.
- Added structured inventory parsing for YAML, JSON and constrained CSV, with host normalisation, deterministic deduplication and source provenance.
- Added inventory-backed discovery and audit examples, goldens and report rendering so imported owner, environment, tags, notes, provenance and imported ports are visible in the shipped output contract.

## Security

No known security fixes in this release.

## In This Release

- This release keeps the remote model narrow: `--inventory-file` extends the existing `remote` commands rather than adding connectors, cloud APIs, a database or broader scanner behaviour.
- Inventory manifests may declare per-entry ports, and `--ports` remains available as a run-level override. The simple line-based host contract stays on `--targets-file`.
- Platform-specific import adapters remain later work. Surveyor still emits one canonical JSON output plus derived Markdown rather than vendor-shaped output formats.
