Surveyor v0.3.1 is a maintenance release focused on internal clarity. It does not change the shipped audit, discovery or explicit-target TLS behaviour, but it makes the codebase and core docs easier to understand, review and extend safely.

## Highlights

- Clarified internal contracts across the config, core, discovery, audit, scanner and output packages with higher-signal doc comments and invariant comments.
- Tightened the architecture and audit docs so the current local-audit runner, selection rules and discovery-to-scan boundary are easier to follow.
- Preserved the current CLI and report behaviour while improving maintainability for future work.

## Security

No known security fixes are included in this release.

## In this release

- This release is intentionally narrow. It does not change the current `surveyor audit local`, `surveyor discover local` or `surveyor scan tls` behaviour.
- The current JSON-first, Markdown-derived reporting model is unchanged.
- The main value is stronger internal documentation and clearer invariants around the existing implementation.
