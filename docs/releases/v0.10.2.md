Surveyor v0.10.2 is a release-hardening patch. It does not widen the product surface, but it raises the standard of verification around the code that is already shipped and makes the project's testing and remediation expectations explicit.

## Highlights

- Surveyor now has dedicated static-analysis and fuzzing workflows in the same split-CI style as the rest of the repository, with `staticcheck`, `gosec` and bounded fuzz targets running continuously rather than being left to manual discipline.
- The release workflow itself is now stricter: before tagging and publishing, it runs `staticcheck`, `gosec`, `govulncheck` and short fuzz smoke checks against the parser-heavy packages.
- The repository now documents its test-addition and vulnerability-remediation policy directly in the contributor and security guidance, so the project's engineering expectations are explicit rather than implicit.

## Security

No known security fixes in this release.

## In this release

- Surveyor now pins the Go toolchain to `go1.26.2`, which keeps the release analysis surface aligned with the patched standard-library line currently reported by `govulncheck`.
- Parser and normalisation paths now include debug-only assertions during fuzzing, and the repository carries explicit rationale for the intentional trust-boundary cases that static analysis would otherwise flag without context.
- This release also carries the recent repository-surface improvements, including the Surveyor logo and Go Report Card badge, but the substance of the release is engineering hardening rather than feature expansion.
