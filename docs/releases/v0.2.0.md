Surveyor v0.2.0 makes discovery a first-class part of the tool. The repository no longer stops at explicit TLS targets only; it can now inventory local listening or bound endpoints and describe them in the same JSON-first, Markdown-derived style as the existing TLS slice.

## Highlights

- Added `surveyor discover local`, which enumerates local TCP listening endpoints and UDP bound endpoints and emits Markdown and optional JSON output.
- Added discovery-side endpoint reporting with best-effort process metadata and conservative low-confidence protocol hints, while keeping hints clearly separate from verified scans.
- Brought the repository docs, examples, safety model and release checklist into line with the implemented discovery foundation so the shipped scope is explicit and defensible.

## Security

No known security fixes are included in this release.

## In this release

- Discovery is local only and observational only. It does not perform active probing, verified protocol identification or automatic scan handoff.
- The explicit-target TLS inventory path remains available under `surveyor scan tls` and its current collection and classification behaviour is unchanged by this release.
- Discovery output follows the same contract discipline as the TLS slice: canonical JSON first, derived Markdown second.
