Surveyor v0.10.1 is a focused maintenance release. It tightens the repository's engineering surface rather than widening the product, chiefly by landing the new split CI workflow suite and removing the remaining Go Report Card gocyclo regressions.

## Highlights

- The repository now uses a fuller CI surface instead of one broad workflow, including dedicated Go test, lint, build, module-hygiene, vulnerability, release-verification, workflow-validation, script-quality, action-drift and benchmark checks.
- The Go Report Card gate is now treated as checked-in repository logic rather than opaque workflow glue, with explicit helper scripts, parser tests and hardened network policy around the external service call.
- The last high-complexity hotspots in the main remote parsing, report validation, Markdown rendering and test paths were refactored so Surveyor now clears the current `gocyclo > 15` bar cleanly.

## Security

No known security fixes in this release.

## In This Release

- GitHub Actions hardening is now stricter and more operationally correct. The imported CI workflows and release workflow use explicit block-mode `step-security/harden-runner` allow-lists, including the GitHub-hosted runner DNS path required for reliable outbound resolution.
- This release does not add new scanners, report kinds or adapter breadth. It is a stability and maintainability release, with the practical outcome that CI is stronger and the codebase is easier to review.
- The README also now includes a Discussions section so the public repository surface better reflects how project conversation should happen.
