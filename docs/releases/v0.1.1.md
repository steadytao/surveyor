Surveyor v0.1.1 is a small tightening release over the initial TLS inventory MVP work. It does not expand scope. It makes the released surface more coherent and reduces release-maintenance friction.

## Highlights

- The primary example target file is now aligned with the checked-in example reports so the first documented scan path reads like a clean happy-path example instead of a partially broken one.
- The README now uses an explicit local build output name, calls out the Windows invocation form and avoids claiming unpublished releases.
- Markdown output now reads more like a report and less like API-shaped output by removing unnecessary backticks from ordinary values.
- The manual release workflow now appends the commit changelog automatically so release-note files can stay editorial instead of duplicating commit history by hand.

## Security

No known security fixes are included in this release.

## In this release

- No scanner behaviour was widened in this release.
- The TLS inventory path, classification behaviour and output contract remain unchanged.
- The release process remains manual but release notes now only need the human-written summary sections. The workflow appends the commit changelog automatically.
