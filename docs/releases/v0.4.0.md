Surveyor v0.4.0 is the first remote inventory release. It extends the existing discovery, audit and TLS inventory model across explicitly declared CIDR scope without widening Surveyor into a general network scanner.

## Highlights

- Added `surveyor discover subnet` and `surveyor audit subnet` for scoped remote inventory within explicitly declared CIDR scope and explicit port sets.
- Kept the remote audit path conservative by reusing the existing TLS scanner only, with discovered facts, hints, selection decisions, verified TLS results and skip reasons kept separate.
- Hardened the remote report contract so discovery and audit outputs now carry explicit scope metadata, and remote runs also record the effective execution settings that shaped the run.

## Security

No known security fixes in this release.

## In This Release

- This release is intentionally narrow. Remote scope is currently CIDR-only with explicit `--ports` required and cautious defaults for host caps, concurrency and timeout.
- Remote dry-run support remains plan-only output, not canonical report JSON so users can validate intended scope without generating fake observed results.
- Remote IP-target TLS results should be read as literal connection-path observations. This release does not add hostname validation, trust validation or full virtual-host coverage claims.
