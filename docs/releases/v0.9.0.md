Surveyor v0.9.0 adds the first platform-specific import adapter layer on top of the existing inventory, discovery, audit, diffing and workflow surface. Surveyor can now ingest stable Caddy and Kubernetes sources into its canonical imported-inventory model without turning the product into a connector-heavy inventory platform.

## Highlights

- Added platform-specific import adapters to the canonical remote command surface, including `caddy` for Caddy JSON and Caddyfile input plus `kubernetes-ingress-v1` for Kubernetes Ingress v1 manifests.
- Added explicit adapter selection and preserved adapter provenance in the shipped output surface, including `--adapter`, `--adapter-bin`, `scope.adapter` and `adapter_warnings`.
- Kept the adapter layer disciplined: Caddyfile support now uses external `caddy adapt` rather than embedding Caddy's module graph into the Surveyor binary, and Kubernetes support remains intentionally scoped to Ingress v1 rather than generic cluster parsing.

## Security

No known security fixes in this release.

## In this release

- `Caddyfile` and `*.caddyfile` now auto-detect the `caddy` adapter, while explicit `--adapter caddy` also works for valid Caddyfiles stored under non-standard names such as `site.conf`.
- Adapter-backed discovery and audit reports now render adapter provenance and adapter warnings in both canonical JSON and derived Markdown, and the repository includes checked-in Caddy and Kubernetes examples and goldens for that surface.
- Surveyor still does not add live cloud or CMDB connectors, a second import command family or a generic Kubernetes parser in this milestone, and Caddyfile support relies on an external Caddy executable.
