Surveyor v0.5.0 makes remote scope first-class. It keeps the existing conservative TLS-only inventory model, but generalises the remote command surface so authorised scope can come from either explicit CIDR ranges or simple approved host lists.

## Highlights

- Added canonical `surveyor discover remote` and `surveyor audit remote` commands, while retaining `discover subnet` and `audit subnet` as CIDR-only compatibility aliases during `v0.5.x`.
- Added simple file-backed remote scope through `--targets-file`, keeping `--ports` explicit and preserving the current discovery, hinting, selection and verified-scanning boundaries.
- Updated the repository docs, examples and golden outputs so the shipped remote surface is described honestly and exercised with both the canonical command family and file-backed scope metadata.

## Security

No known security fixes in this release.

## In this release

- This release keeps TLS as the only deep verified remote scanner. It does not add STARTTLS, non-TLS deep scanners, trust validation or hostname validation semantics.
- File-backed scope is intentionally simple: one host or IP per line, blank lines and `#` comments allowed, no inline ports and explicit `--ports` still required.
- Remote dry-run output remains an execution plan rather than canonical JSON so scope can be validated without generating fake observed or verified results.
